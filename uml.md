CLIPilot Agent Loop 深度分析（MainAgent 架构）

> **架构更新 (2026-02-26)**: goal-driven-execution 重构已完成。关键变更：
> - **删除**: Planner, TaskGraph, Scheduler — 不再有多阶段规划/调度
> - **新流程**: `Goal → MainAgent.executeGoal(goal) → GoalResult`
> - MainAgent 现在是目标驱动的无界循环，由 LLM 自主决定何时完成
> - SignalRouter 新增执行控制: `pause()`, `resume()`, `abort()`
> - MainAgent 事件: `goal_start`, `goal_complete`, `goal_failed`, `need_human`, `log`
> - 工具变更: 移除 `request_replan`, 总计 12 个内置工具
> - ContextManager 模块: 移除 `task_graph_summary`
>
> 以下图表为重构前的历史架构参考。

这是一个 元级 Agent 编排系统 —— 用一个 LLM（MainAgent）通过 tool use 循环来监控和控制另一个 Agent（Claude Code），通过 tmux 实现非侵入式交互。

关键发现总结

┌────────────────┬───────────────────────────────────────────────────────────┐
│    特征         │                           描述                           │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 架构模式       │ 元编排器 (Meta-Orchestrator)，用 LLM tool use 控制       │
│                │ LLM Agent，MainAgent 为中枢决策节点                       │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 通信方式       │ 通过 tmux 桥接，非侵入式（Agent 不知道自己被控制）       │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 决策方式       │ LLM tool use 循环：complete → 提取工具调用 → 执行 →      │
│                │ 返回结果 → 继续，直到终止性工具或无工具调用               │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 信号路由       │ 分层：快速通道（高置信度跳过/自动完成）+                  │
│                │ MainAgent 通道（需要推理的信号）                          │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 状态检测       │ 三层：正则快速匹配 → LLM 语义分析 → 深度错误诊断        │
│                │ 输出仅 status/confidence/detail，不含 suggestedAction     │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 上下文管理     │ 模块化系统提示 + 持续对话历史 + 基于 token 阈值的压缩    │
│                │ 70% 上下文窗口 → 触发 LLM 压缩 → 注入 compressed_history │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 错误恢复       │ MainAgent 自主决策：mark_failed / request_replan /       │
│                │ escalate_to_human，基于完整上下文推理                     │
├────────────────┼───────────────────────────────────────────────────────────┤
│ Spec 可扩展性  │ MainAgent 可注入 /opsx 命令，SignalRouter 自适应扩展      │
│                │ tmux 抓取行数 (50→300)，fetch_more 工具作为兜底           │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 工具系统       │ 7 个工具：3 非终止性 (send/respond/fetch) +              │
│                │ 4 终止性 (complete/failed/replan/escalate)                │
└────────────────┴───────────────────────────────────────────────────────────┘
